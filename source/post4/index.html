The p5.js Project

Besides being a javascript library, p5.js is a community open-source project. They receive support from the processing foundation, which is supported by donations of users and people interested in the project. I'd like to quote a short sentence from the home page linked above:

We are all learners.

The p5.js Library

From the static point of view, the p5.js library is a tool for drawing shapes. As in a canvas. We can though, achieve a sort of dynamic, living canvas. Here I'll be taking a look at the very basics.

setup and draw

                       
                        function setup() {

                                 }

                        function draw() {

                                }

setup and draw are the two main functions of the program. In both we can write plain javascript, or use some other built-in features. setup controls static objects, draw controls the dynamic part. Once we wrote all the code in setup, this is kindof the base of our work.

                       
                        function setup (){
                        createCanvas(300, 400);
                        background(51); // 0-255 --> greyscale.
                        }

                        function draw() {

                        }

If we know nothing of programming, the words are telling us:

  1. create a canvas 300pixels horizontal, 400pixels vertical;
  2. background(51) is background's color in greyscale, 0 being black, 255 being white.

Shapes

p5.js comes with tools for easy drawing shapes. We can draw rectangles, ellipses, cubes and many more. Let's go ahead and draw a couple of shapes here and there. All of them will be part of our Bob Ross-like static canvas. 

                        
                    function setup(){ 

                       let x=200, y=180;
                       createCanvas(x,y, WEBGL); //We need WEBGL for 3D-like shapes
                       background(51, 25, 128); //rgb values.
                       box(15, 15, 15) 
                       stroke("red"); //string color values
                       fill(255,0,0, 60) //rgba. The last num is %opacity.
                       rect(0,0,60,40);

                       stroke("#00FF00")//green hex value
                       fill(0,255,0, 40)
                       // noStroke() removes borderline (black by default), 
                       //but it looks ugly, so I commented it out.
                       rect(30, 30, 60, 40);  

                       }

                       function draw(){

                       }

Many conclusions can be drawn out of this example. First, there are many ways to specify colors. For starters we can stick to one. The most general of those is rgba, so we'll mostly use that one. These are all properties taking color as an argument:

  • background()
  • fill()
  • stroke()
they are self explanatory.

By default shapes are filled up with white. Some functions like fill, stroke, noStroke only affect the code coming next, which is very handy. (the correct capitalization in the name is essential).

Rectangles' left corner is by default on page's center. The function takes 4 values; first 2 being the left corner's distance from the center, the other 2 are width and height. We'll see how to change this behaviour to a more intuitive one later.

On the other hand, Cubes take only 3 values but they start from the center (and split evenly).

Last but not least, we can use the noThis functions, named in camelCase way. As most of the other functions, they will only affect subsequent code. An example is noStroke function. There are noFill, noErase, and probably many more.

Where do I take all this data from

All libraries should have a page explaining how are we supposed to use it. And so has p5.js; there is a reference page, that we will consult to understand what arguments can we pass to the functions, and are they supposed to do, and to grab examples!

What we know now

  • there are 2 main functions: setup and draw.
  • the static pieces are specified in setup, dynamics in draw.
  • background, fill, stroke take a color as argument.
  • Functions affect only subsequent code
  • We can draw many shapes.
  • We can consult the reference page for p5 library.
