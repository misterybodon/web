Brief

On this topic we will talk about how devices transfer data to each other, in what conditions does this happen, and how to protect our privacy.

Data Transfer Between Devices

There is one thing we're not aware of and rules the world. Data. I will assume a scoped notion of data: a file with some content inside. It could be a file.pdf or anything else.

Let's take a concrete example. We open up a browser. We type an URLduckduckgo.com. Automatically, this is translated by the browser to https://duckduckgo.com. HTTPS means Secure Hypertext Transfer Protocol, and it is a way two computers can transfer something called Hypertext. In turn, this is images, text files, binaries. We'll discuss the Secure, later on.

Data is being transferred between two devices through HTTPS.

Internet of Things: Technologies

IoT is not only about protocols (like HTTP), it is way broader. I will go for the easier way to get started, which is tagged Technologies.

Protocols

Protocols relate to specific rules in which vendors can implement a technology.

Not all protocols are under only one technology

Let's list a few protocols

A few common Protocols on the Internet:
HTTP: Hyper-Text Transfer Protocol
HTTPS: HTTP Secure
FTP: File Transfer Protocol
SFTP: Secure FTP
BT: BitTorrent

The secure protocols are very important. If we buy a product in some site, and we do not use a secure protocol, then hackers can see in plain text our user and password. Browsers like Firefox and Chrome force HTTPS in sites. This can also be achieved by a Proxy server. Yet a proxy server does not hide the IP, and that's why VPNs are common today
